Privacy Policy

Effective Date: February 28, 2026 · Version 2.0

1. Introduction

Football Coach (hereinafter "the Application" or "the App") is a mobile application developed to help football coaches manage training sessions, players, and matches. This Privacy Policy explains how the Application collects, uses, discloses, and protects users' personal data.

Developer: Kal-ix
Platforms: Android (Google Play Store)

2. Information Collected

2.1 Data Entered Directly by the User

  • Player names (surname and first name)
  • Players' dates of birth
  • Player photographs
  • Performance data (minutes played, goals, assists, yellow cards, red cards)
  • Training information (date, time, exercises, notes)
  • Match information (date, time, opponent, results, squad members, formation on the pitch)
  • Team name

2.2 Account Data (Google Sign-In)

When the user chooses to log in via Google Sign-In, the following data is collected:

  • Email address of the Google account
  • Display name of the Google account
  • Profile photo of the Google account (URL)
  • Unique user identifier (Firebase UID)

This data is stored in the user profile on Google Cloud Firestore to enable authentication and cross-device synchronization. Login is optional: the App works without authentication, in a fully local mode.

2.3 Cloud-Synced Data (optional feature)

When the user enables cloud synchronization, the following data is uploaded to Google Cloud Firestore:

  • Player data (names, dates of birth, statistics, photographs encoded in base64)
  • Training data (date, time, exercises, notes, exercise images)
  • Match data (date, time, opponent, results, squad members, formation)
  • Exercise data (shared exercise library, including images)
  • Team information (name, owner, members, roles, invitations)

Cloud synchronization is optional and requires Google login. The user can use the App in a fully offline mode without sharing any data to the cloud.

2.4 Automatically Collected Data

App usage data via Google Firebase Analytics:

  • App opening events
  • Features used
  • Session duration
  • App and operating system version
  • Device information (brand, model)

Advertising data via Google Mobile Ads (AdMob):

  • Anonymized advertising identifiers
  • Types of ads displayed
  • Ad interaction data (not linked to personal identities)

Device file access data:

  • Photo and gallery access permissions
  • External storage read/write permissions

3. Legal Basis for Processing

For the EU (GDPR):

  • Explicit consent: The user provides consent upon first launch of the App and upon enabling cloud synchronization
  • Legitimate interest: Improving the App and analyzing usage
  • Contract fulfillment: Operating the App as a requested service

For California (CCPA/CPRA):

  • Collection with prior user consent
  • User rights as described in Section 9

4. Processing Purposes

Personal data is used for:

Application Operation

  • Storage and management of player data
  • Recording training sessions and matches
  • Viewing and exporting PDF reports

Authentication and User Profile

  • User identification via Google Sign-In
  • Creation and management of user profile on Firestore
  • Login session management

Cloud Synchronization and Team Sharing

  • Synchronization of team data across the same user's different devices
  • Sharing the team with other coaches (via email invitation)
  • Management of team member roles and permissions (owner, editor, viewer)
  • Real-time synchronization of changes across all connected devices

Analytics and Service Improvement

  • Analysis of App usage
  • Identification of technical issues
  • Planning new features
  • Understanding user behavior

Personalized Advertising (AdMob)

  • Displaying relevant advertisements
  • Measuring campaign effectiveness
  • Providing insights to advertising partners

Regulatory Compliance

  • Fulfilling legal obligations
  • Protecting rights and security

5. Data Storage and Protection

5.1 Storage

  • Local data: Player, training, and match data are stored locally on the user's device using the Hive database
  • Cloud (Firebase): The following data is sent to Google Firebase:
    • Firebase Analytics: Usage data and crash reporting
    • Firebase Authentication: Google account login data (email, name, profile photo)
    • Cloud Firestore: When synchronization is enabled, team data (players, training sessions, matches, exercises) is stored on Firestore. Photographs are encoded in base64 format and saved directly in Firestore documents (Firebase Storage is not used)
  • No personal server: The Application does not manage private servers for storing personal data. All cloud data is managed through Google Firebase infrastructure

5.2 Protection

  • Local encryption: Data is protected by the device's operating system
  • Encryption in transit: All communications with Firebase (Authentication, Firestore, Analytics, AdMob) use HTTPS/TLS
  • Secure authentication: Login is performed via Google's OAuth 2.0; the App does not handle or store passwords
  • Firestore security rules: Access to cloud data is protected by security rules that restrict reading and writing to authorized users only (team members)
  • Optional cloud data: Personal data remains exclusively on the local device if the user does not enable cloud synchronization

5.3 Retention Period

  • Local data: Retained for the duration of the App installation on the device
  • Firestore data: Cloud-synced data is retained until the user manually deletes it or deletes their account
  • Firebase user profile: Retained until account deletion or permanent logout
  • Analytics: Google retains analytics data for 2-14 months according to their policies
  • AdMob: Google retains advertising data for reporting and service improvement purposes

6. Data Sharing

6.1 Data Shared with Third Parties

The Application shares data with the following services:

Google Firebase Authentication

  • What: Email, display name, profile photo of the Google account
  • Why: User authentication and login session management
  • Privacy Link: firebase.google.com/support/privacy

Google Cloud Firestore

  • What: User profile, team data (players, training sessions, matches, exercises), member information and invitations. Player photographs and exercise images are encoded in base64 and saved in Firestore documents
  • Why: Data synchronization across devices and team sharing between coaches
  • Privacy Link: firebase.google.com/support/privacy

Google Firebase Analytics

  • What: Session identifiers, crash logs, anonymized usage data, device type, OS version
  • Why: Reliability monitoring, diagnostics, usage statistics, feature improvement
  • Privacy Link: firebase.google.com/support/privacy

Google Mobile Ads (AdMob)

  • What: Anonymized advertising identifier, ad interaction data
  • Why: Display personalized ads and measure performance
  • Privacy Link: policies.google.com/privacy

6.2 Sharing Between Users (Shared Team Feature)

When the user enables team sharing:

  • Team data (players, training sessions, matches, exercises) becomes accessible to all invited members
  • The user's email and display name are visible to other team members
  • The role (owner, editor, viewer) determines data modification permissions
  • Invitations contain the recipient's email and the inviter's name

6.3 Data NOT Shared (Offline Mode)

If the user does not log in and does not enable cloud synchronization:

  • Player photographs: Remain ONLY on the local device
  • Player names and personal data: Remain ONLY on the local device
  • Training and match data: Remain ONLY on the local device

6.4 International Transfers

Google (Firebase, Firestore, and AdMob) transfers data to the USA and other countries. Google services are certified according to:

  • Standard Contractual Clauses (SCC)
  • Privacy Shield (where applicable)

  • Right to deletion: Request deletion of data

  • Right to correction: Request correction of inaccurate data
  • Right to restriction: Limit use of sensitive data

Contact: To exercise these rights, contact: request.kalix@gmail.com

7.3 Cloud Data Deletion

The user may at any time:

  • Log out to stop cloud synchronization
  • Delete the team from the cloud to remove all synced data from Firestore
  • Leave a shared team to remove their access to the data
  • Delete their account directly from the app using the “Delete account and cloud data” option in the side menu. This operation deletes:
    • All teams owned by the user (including players, training sessions and matches)
    • Membership in shared teams
    • Pending invitations
    • The user profile from Firestore
    • The Firebase Authentication account
  • Alternatively, users may request complete deletion by contacting the developer at request.kalix@gmail.com

The Application does NOT explicitly collect the following sensitive data:

  • Health data

Note: Player photographs are images that are part of the App's ordinary functionality and are not treated as "biometric data" unless used for facial recognition (which the App does not do). When cloud synchronization is enabled, photographs are encoded in base64 and uploaded to Firestore to enable cross-device synchronization.

11. Cookies and Tracking Technologies

11.1 Use of Cookies

The Application does NOT use traditional cookies (it is not a web app). However:

  • Firebase and Google Analytics may use local persistent identifiers
  • Firebase Authentication stores session tokens to keep the user logged in

11.2 Advertising Identifiers

  • Android (Google Advertising ID): An anonymized identifier provided by Google
  • iOS (IDFA - Identifier for Advertisers): An anonymized identifier provided by Apple

These cannot be correlated to the user's identity without additional information.

12. Data Security

The Application implements the following security measures:

  1. Encrypted local storage: Data on the device is protected by the operating system
  2. HTTPS/TLS connections: All communications with cloud services use encryption
  3. OAuth 2.0 authentication: Login is performed via Google's secure protocol; no passwords are handled or stored by the App
  4. Firestore security rules: Cloud data access is restricted to authorized team members only, with role-based permissions
  5. No hardcoded credentials: All secrets are securely managed
  6. Security updates: Regular patches for known vulnerabilities
  7. Optional synchronization: Personal data remains on the local device if the user does not choose to enable cloud synchronization

Limitation: No system is completely secure. Users are responsible for their device security.

13. Protection of Minors

The Application is NOT intended for children under 13 years old. We do not intentionally collect personal data from minors.

However, since the Application manages data about young players (such as photos and dates of birth), parents/guardians must:

  • Provide appropriate consent if residing in the EU (GDPR)
  • Understand that player data remains local on the coach's device, unless cloud synchronization is enabled
  • Understand that, if cloud synchronization is enabled, player data (including photos and dates of birth) is uploaded to Google Cloud Firestore
  • Configure the device with parental controls if appropriate
